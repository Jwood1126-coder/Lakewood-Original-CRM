Setup checklist (Google Cloud Console)
  1. Go to console.cloud.google.com → APIs & Services → Credentials
  2. If you don't have a project yet, create one (e.g. "Lakewood CRM")
  3. Enable the Gmail API: API Library → Gmail API → Enable
  4. Configure the OAuth consent screen:
    • User Type: External (unless you're on Google Workspace, then Internal)
    • Add yourself as a Test user (avoids the 100-user verification process while you're the only user)
    • Scopes: add .../auth/gmail.readonly and .../auth/userinfo.email
  5. Create credentials → OAuth client ID → Application type: Web application
  6. Set Authorized redirect URIs to:
    {{ url_for('gmail.callback', _external=True) }}
  7. Copy the generated Client ID and Client secret
  8. In Railway → Variables, set GMAIL_CLIENT_ID and GMAIL_CLIENT_SECRET
  9. Wait ~1 min for Railway redeploy, then come back here and click Connect to Gmail
Forwarding business mail from Outlook (Microsoft 365) to this Gmail
  1. Sign in at outlook.office.com with your business account
  2. Settings (gear icon) → View all Outlook settings → Mail → Forwarding
  3. Enable forwarding, set the destination to your Gmail address
  4. Check Keep a copy of forwarded messages so business mail still archives in M365
  5. Save

Note: Outlook forwarding rewrites the From header to your business address, hiding the original sender. The Phase 2 parser will dig the original sender out of the forwarded body. If you want cleaner sender preservation today, set up a "redirect" rule via Outlook's mail rules instead of plain forwarding — same destination, but the original From survives.
